2.16 - Expressions de fonctions

Concepts clés

Function Declaration 
function sayHi() {
    alert("Hi");
}
Function Expression 
let sayHi = function() {
    alert("Hi");
};
  • Hoisting : les déclarations sont "hissées" — utilisables avant leur définition dans le code. Les expressions ne le sont PAS.
  • Scope de bloc : une Function Declaration dans un bloc { } n'est visible QUE dans ce bloc (en mode strict)
  • Callbacks : passer une fonction en argument à une autre fonction pour qu'elle soit appelée plus tard
  • Fonctions anonymes : expressions sans nom, souvent utilisées comme callbacks
  • Les fonctions sont des valeurs — on peut les assigner, les passer, les retourner
FONDAMENTAL : les callbacks sont la base de la programmation asynchrone en JavaScript. Comprendre "passer une fonction en argument" est essentiel.

Exercices

Ex1 — Quelle est la différence ?

Observe ces deux codes. L'un fonctionne, l'autre génère une erreur. Lequel ? Pourquoi ?

// Code A
sayHi(); // Appelé AVANT la définition
function sayHi() {
    console.log("Hi from declaration!");
}

// Code B
greet(); // Appelé AVANT la définition
let greet = function() {
    console.log("Hi from expression!");
};

Code A : fonctionne / erreur ?

Code B : fonctionne / erreur ?

Le "hoisting" (levée) est ce qui différencie les deux. Cherche ce terme si tu ne le connais pas encore.

Ex2 — Pourquoi ce code échoue-t-il ?

Identifie pourquoi ce code ne fonctionne pas et propose une correction :

let age = 20;

if (age >= 18) {
    function welcome() {
        console.log("Bienvenue !");
    }
} else {
    function welcome() {
        console.log("Accès refusé.");
    }

welcome(); // Erreur en mode strict !

Ex3 — Passer une fonction en callback

Écrire une fonction doOperation(a, b, operation) qui prend deux nombres et une fonction operation, et retourne le résultat de operation(a, b).

function add(x, y) { return x + y; }
function multiply(x, y) { return x * y; }

doOperation(3, 4, add);       // → 7
doOperation(3, 4, multiply);  // → 12
doOperation(10, 2, function(x, y) { return x - y; }); // → 8 (fonction anonyme inline)

La fonction operation est un callback — on ne l'appelle pas quand on la passe, on la passe comme valeur. Note l'absence de () lors du passage.

B3 — Mini-système d'événements

Implémenter un système simple d'événements avec :

  • on(eventName, callback) — enregistre un callback pour un événement
  • emit(eventName, data) — déclenche tous les callbacks de cet événement
on("login", function(user) { console.log("Bienvenue", user); });
on("login", function(user) { console.log("Log: connexion de", user); });
on("logout", function(user) { console.log("Au revoir", user); });

emit("login", "Alice");
// Bienvenue Alice
// Log: connexion de Alice

emit("logout", "Alice");
// Au revoir Alice

Utilise un objet {} comme "registre" d'événements. Chaque clé est un nom d'événement, sa valeur est un tableau de callbacks.
