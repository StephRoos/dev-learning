Ex3 — Réécrire avec les bonnes pratiques

Ce code fonctionne mais est mal écrit. Améliore-le en appliquant les bonnes pratiques vues dans les chapitres 2.1-2.17.

// Code à améliorer :
var x = prompt("enter age");
var y;
if (x >= 18) {
    y = true;
} else {
    y = false;
}

var msg;
if (y == true) {
    msg = "adult";
} else if (y == false) {
    msg = "minor";
}

for (var i = 0; i < 3; i = i + 1) {
    if (i == 0) {
        console.log("zero");
    } else if (i == 1) {
        console.log("one");
    } else if (i == 2) {
        console.log("two");
    }
}

alert(msg);

Points à améliorer :

  • Utiliser let/const au lieu de var
  • Nommer les variables avec des noms significatifs
  • Simplifier les conditions (if (y == true) est redondant)
  • Utiliser un ternaire pour les assignations simples
  • Remplacer le if/else if répétitif par un switch
  • Utiliser += 1 → ++

Bonus — Mini-projets

B1 — Calculatrice en console

Créer une calculatrice complète qui combine toutes les notions vues :

  • Demander le premier nombre, l'opérateur (+, -, *, /), le deuxième nombre via prompt
  • Valider les entrées (nombres valides, pas de division par zéro)
  • Calculer avec switch
  • Afficher le résultat proprement avec des template literals
  • Proposer de recommencer (boucle + confirm)

Structure : boucle do..while + fonctions séparées pour la validation et le calcul.

B2 — Jeu de devinette de nombre

Créer un jeu complet :

  • Le programme choisit un nombre entre 1 et 100 (Math.floor(Math.random() * 100) + 1)
  • L'utilisateur a 7 essais maximum
  • Après chaque essai : indiquer "Trop petit", "Trop grand" ou "Bravo !"
  • À la fin : afficher le nombre d'essais utilisés
  • Proposer de rejouer

Utilise une boucle for (nombre d'essais), if/else pour les indices, et une fonction séparée pour le jeu principal.

generatePassword(12, { uppercase: true, numbers: true, symbols: false })
// → "aBcDeFgH1234" (exemple)
