2.15 - Fonctions

Concepts clés

  • Déclaration : function nom(parametre1, parametre2) { ... }
  • Paramètres vs arguments : paramètres = variables locales dans la fonction, arguments = valeurs passées à l'appel
  • Valeur par défaut : function nom(param = "défaut") { } — utilisée si l'argument est undefined
  • return : arrête l'exécution de la fonction et retourne une valeur. Sans return → retourne undefined
  • Variables locales : déclarées dans une fonction, invisibles de l'extérieur
  • Nommage : utiliser des verbes d'action (getAge, calcSum, showMessage). Une fonction = une action.
FONDAMENTAL : une fonction devrait soit retourner une valeur, soit produire un effet de bord (afficher, modifier) — pas les deux en même temps. Ça facilite les tests et la lisibilité.

Exercices

Ex1 — Le "else" est-il nécessaire ?

Ces deux fonctions font-elles exactement la même chose ?

// Version A
function checkAge(age) {
    if (age > 18) {
        return true;
    } else {
        return false;
    }
}

// Version B
function checkAge(age) {
    if (age > 18) {
        return true;
    }
    return false;
}

Oui ou non ? Explique pourquoi.

Quand return est atteint, la fonction s'arrête immédiatement. Le code après est-il jamais atteint ?

Ex2 — Réécrire avec '?' ou '||'

Réécrire ces fonctions de manière plus concise :

// À réécrire avec ?:
function checkAge(age) {
    if (age > 18) {
        return true;
    } else {
        return false;
    }
}

// À réécrire avec ||
function checkAge(age) {
    if (age > 18) {
        return true;
    } else {
        return confirm("Did parents allow you?");
    }
}

Ex3 — Fonction min(a, b)

Écrire une fonction min(a, b) qui retourne le minimum de deux nombres.

min(2, 5)  // → 2
min(3, -1) // → -1
min(1, 1)  // → 1

Ex4 — Fonction pow(x, n)

Écrire une fonction pow(x, n) qui retourne x élevé à la puissance n (x^n).

Contrainte : utiliser une boucle (pas Math.pow ni **).

pow(3, 2)  // → 9
pow(3, 3)  // → 27
pow(1, 100) // → 1

Multiplier x par lui-même n fois.

Bonus

B1 — Conversions Celsius / Fahrenheit

Écrire deux fonctions :

  • celsiusToFahrenheit(c) : formule = (c × 9/5) + 32
  • fahrenheitToCelsius(f) : formule = (f - 32) × 5/9
celsiusToFahrenheit(0)    // → 32
celsiusToFahrenheit(100)  // → 212
fahrenheitToCelsius(32)   // → 0
fahrenheitToCelsius(98.6) // → 37

B2 — Vérifier si un nombre est premier

Écrire une fonction isPrime(n) qui retourne true si n est un nombre premier.

isPrime(2)   // true
isPrime(3)   // true
isPrime(4)   // false (2 × 2)
isPrime(17)  // true
isPrime(1)   // false (convention : 1 n'est pas premier)

Un nombre premier n'a aucun diviseur entre 2 et √n.

B3 — Somme récursive de 1 à n

Écrire une fonction sumTo(n) qui calcule 1 + 2 + ... + n.

Implémenter 3 versions :

  • Version avec boucle for
  • Version récursive : sumTo(n) = n + sumTo(n-1)
  • Version formule mathématique : n * (n + 1) / 2
sumTo(1)  // → 1
sumTo(5)  // → 15 (1+2+3+4+5)
sumTo(100) // → 5050

La récursion est un fondamental à comprendre. Pense à un "cas de base" pour arrêter la récursion (n === 1).
