2.13 - Boucles : while et for

Concepts clés

  • while (condition) { ... } — vérifie la condition AVANT chaque itération
  • do { ... } while (condition) — exécute le bloc AU MOINS UNE FOIS, vérifie après
  • for (init; condition; step) { ... } — la boucle la plus courante
  • break — sort immédiatement de la boucle
  • continue — passe à l'itération suivante (saute le reste du bloc)
  • Labels : outer: for (...) permet de break outer depuis une boucle imbriquée
  • Attention : ++i incrémente PUIS retourne la valeur. i++ retourne la valeur PUIS incrémente.

Ex1 — Dernière valeur de la boucle

Quelle valeur a i après cette boucle ?

    i--;
}
// i vaut ???

Trace les itérations : i=3 (truthy → continue), i=2, i=1, i=0 (falsy → stop).

Ex6 — Répéter jusqu'à la bonne saisie

Demander un nombre avec prompt. Répéter jusqu'à ce que l'utilisateur saisisse un nombre supérieur à 100 (ou annule).

Utiliser do..while — parfait ici car on veut poser la question au moins une fois.

do { ... } while (condition) — la condition est vérifiée APRÈS le premier passage.

Ex7 — Afficher les nombres premiers jusqu'à n

Afficher tous les nombres premiers de 2 à n (utilise n = 20).

Un nombre premier n'est divisible que par 1 et par lui-même.

Algorithme : pour chaque nombre, essaie de le diviser par tous les entiers de 2 à √nombre. Si aucun ne divise, c'est un premier. Utilise break et un label pour sortir proprement.

B3 — PGCD avec while (algorithme d'Euclide)

Calculer le Plus Grand Commun Diviseur (PGCD) de deux nombres.

Algorithme d'Euclide : tant que b != 0, faire a, b = b, a % b.

pgcd(48, 18) // → 6
pgcd(100, 75) // → 25
pgcd(7, 13)  // → 1 (nombres premiers entre eux)

En JS : let temp = b; b = a % b; a = temp; ou avec déstructuration [a, b] = [b, a % b].
