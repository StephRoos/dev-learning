2.12 - Opérateur de coalescence des nuls ??

Concepts clés

  • a ?? b retourne a si a n'est ni null ni undefined, sinon retourne b
  • Différence avec || : || ignore toute valeur falsy (0, "", false), ?? n'ignore que null/undefined
  • Cas d'usage principal : fournir une valeur par défaut quand une valeur peut ne pas être définie
  • Priorité basse : s'évalue après la plupart des opérateurs, utilise des parenthèses si besoin
  • ??= : assignation par coalescence — assigne seulement si la valeur actuelle est null/undefined
Valeur valeur || "défaut" valeur ?? "défaut"
null "défaut" "défaut"
undefined "défaut" "défaut"
0 "défaut" 0
"" "défaut" ""
false "défaut" false
"hello" "hello" "hello"

Exercices

Ex1 — null ?? "default"

Que retourne chaque expression ? Prédis avant d'exécuter.

null ?? "default"
undefined ?? "default"
"hello" ?? "default"
0 ?? "default"

Ex2 — Chaînage de ??

Que retourne cette expression ?

undefined ?? null ?? "hello"

Et celle-ci ?

null ?? undefined ?? 0 ?? "fallback"

?? évalue de gauche à droite et retourne le premier non-null/non-undefined.

Ex3 — Différence || vs ?? avec 0, "" et false

C'est l'exercice le plus important ! Prédis le résultat :

// Avec une variable valant 0 :
let userScore = 0;

console.log(userScore || "Score inconnu");  // ???
console.log(userScore ?? "Score inconnu");  // ???

// Avec une chaîne vide :
let userName = "";

console.log(userName || "Anonyme");  // ???
console.log(userName ?? "Anonyme");  // ???

// Avec false :
let darkMode = false;

console.log(darkMode || true);  // ???
console.log(darkMode ?? true);  // ???

Voilà pourquoi ?? existe : un score de 0, un nom vide ou false sont des valeurs intentionnelles qui ne doivent pas être remplacées par un défaut.

Bonus

B1 — userConfig avec valeurs par défaut via ??

Créer un objet userConfig qui peut avoir des propriétés manquantes (null/undefined), et construire une configuration finale avec des valeurs par défaut :

const userConfig = {
    theme: null,       // l'utilisateur n'a pas choisi
    fontSize: 0,       // l'utilisateur veut 0 (taille minimale !)
    language: undefined,
    notifications: false  // l'utilisateur a désactivé
};

Construire finalConfig avec ces défauts : theme="light", fontSize=16, language="fr", notifications=true.

Attention : fontSize=0 et notifications=false sont intentionnels, utilise ?? et non ||.

B2 — Utiliser ??= pour initialiser

??= est un raccourci : a ??= b équivaut à a = a ?? b

Utiliser ??= pour initialiser les propriétés manquantes d'un objet utilisateur :

let user = {
    name: "Alice",
    age: null,
    // role manquant (undefined)
};

Initialiser user.age à 0, user.role à "visitor", sans écraser user.name.

B3 — Système de fallback en cascade

Simuler un système de configuration en cascade : valeur utilisateur → valeur de config → valeur par défaut.

Écrire une fonction getSetting(userValue, configValue, defaultValue) qui retourne la première valeur non-null/non-undefined.

getSetting(null, null, "dark")      // "dark"
getSetting(null, "light", "dark")   // "light"
getSetting("custom", "light", "dark") // "custom"
getSetting(0, null, 16)             // 0 (pas 16 !)
getSetting(false, null, true)       // false (pas true !)
