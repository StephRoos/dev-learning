5.6 — Iterables

Concepts clés : Symbol.iterator

Un objet est itérable s'il implémente la méthode Symbol.iterator. Cette méthode doit retourner un itérateur — un objet avec une méthode next().

// Protocole d'itération :
// Symbol.iterator → retourne un itérateur
// next() → retourne { value: ..., done: false } ou { value: undefined, done: true }

let range = {
    from: 1,
    to: 5,
    [Symbol.iterator]() {       // méthode avec clé symbolique
        let current = this.from;
        let last = this.to;
        return {
            next() {
                if (current <= last) {
                    return { value: current++, done: false };
                } else {
                    return { value: undefined, done: true };
                }
            }
        };
    }
};

for (let n of range) {
    console.log(n); // 1, 2, 3, 4, 5
}

Iterable vs Array-like :

  • Iterable : a Symbol.iterator → utilisable avec for..of
  • Array-like : a des indices numériques et length → ressemble à un tableau mais n'en est pas un
  • Les vrais tableaux sont les deux à la fois

Array.from(iterable) convertit n'importe quel itérable ou array-like en vrai tableau.

Bonus 1 — Itérable infini : suite de Fibonacci

Un itérable infini n'a jamais done: true. Le break dans for..of appelle automatiquement la méthode return() de l'itérateur si elle existe (nettoyage des ressources).

Bonus 2 — Itérer sur les caractères Unicode

Comprends pourquoi for..of gère mieux Unicode que l'accès par index.

for..of sur une string itère sur les points de code Unicode, pas sur les unités de code UTF-16. Crucial pour les emojis et les caractères de langues asiatiques.
