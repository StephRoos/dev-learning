5.12 — Méthodes JSON, toJSON

JSON — JavaScript Object Notation

JSON est un format texte universel pour sérialiser des données (API, stockage, communication entre serveurs).

// Sérialiser : objet JS → string JSON
let json = JSON.stringify(value, replacer, spaces);
// replacer : tableau de clés à inclure, ou fonction de filtrage
// spaces : nombre d'espaces pour indentation (lisibilité)

// Désérialiser : string JSON → objet JS
let obj = JSON.parse(json, reviver);
// reviver : fonction de transformation au parsing

Ce que JSON supporte

// Supporté :
{ "string": "valeur", "number": 42, "bool": true, "null": null,
  "array": [1,2,3], "object": {} }

// PAS supporté (silencieusement ignoré ou erreur) :
// - undefined
// - function
// - Symbol
// - Références circulaires (erreur)
// - Date (converti en string ISO, pas reconverti en Date automatiquement)

toJSON personnalisé

let user = {
    name: "Jean",
    createdAt: new Date(),
    toJSON() {
        // Contrôler ce qui est sérialisé
        return { name: this.name, created: this.createdAt.toISOString() };
    }
};

Fondamental : JSON.parse/stringify est souvent utilisé pour des deep clones rapides, mais avec des limitations importantes (Dates, fonctions, undefined perdus).

Exercice 1 — Sérialiser et désérialiser

Sérialise un objet en JSON puis désérialise-le. Vérifie que l'objet reconstruit est équivalent.

Exercice 2 — Exclure les références circulaires

Utilise le paramètre replacer de JSON.stringify pour gérer une structure avec référence circulaire.

Le replacer peut retourner undefined pour exclure une propriété. C'est la façon standard de contrôler la sérialisation.

Bonus 1 — Sérialiser des Dates correctement

JSON.stringify convertit les Dates en strings, mais JSON.parse ne les reconvertit pas. Implémente un reviver pour restaurer les Dates.

Bonus 2 — Pretty-print JSON

Affiche un JSON formaté avec indentation personnalisée et une ligne de séparation.

Bonus 3 — Deep clone avec JSON et ses limitations

Utilise JSON pour faire un deep clone, puis observe ce qui est perdu.

structuredClone() est disponible nativement depuis 2022 et gère correctement les Dates, Maps, Sets, et même les références circulaires. C'est la solution recommandée pour les deep clones modernes.
