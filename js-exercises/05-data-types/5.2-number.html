5.2 — Nombres

Concepts clés

Notations : 1e9 = 1 milliard, 1e-6 = 0.000001

Bases : 0xff (hex), 0b1010 (binaire), 0o17 (octal)

Arrondis : Math.floor (bas), Math.ceil (haut), Math.round (proche), Math.trunc (entier)

Imprécision flottante : Les nombres sont stockés en IEEE 754 (64 bits). 0.1 + 0.2 !== 0.3

Tests : isNaN(), isFinite(), Number.isNaN() (plus strict)

Parsing : parseInt("12px") → 12, parseFloat("1.5em") → 1.5

// Attention à NaN :
NaN === NaN // false ! NaN n'est égal à rien, même pas lui-même
isNaN("hello") // true (convertit d'abord en nombre)
Number.isNaN("hello") // false (vérifie que c'est NaN sans conversion)

Exercice 1 — Additionner des nombres saisis

Demande deux nombres à l'utilisateur avec prompt, convertis-les en nombres, additionne-les. Gère le cas où l'utilisateur entre une valeur non numérique.

Le + unaire devant une variable (+str) est un raccourci pour convertir en nombre.

Exercice 2 — Pourquoi 6.35.toFixed(1) == 6.3 ?

Ce résultat semble bizarre. Explique en commentaire et trouve comment arrondir correctement.

Fondamental : les nombres flottants ne peuvent pas représenter exactement toutes les valeurs décimales. Toujours tester avant de faire confiance à un arrondi.

Exercice 3 — Boucler jusqu'à saisie valide

Crée une fonction qui redemande un nombre tant que la saisie n'est pas un nombre fini valide.

isFinite(num) retourne false pour NaN, Infinity et -Infinity. C'est le meilleur test pour valider un nombre réel.

Exercice 4 — La boucle infinie occasionnelle

Cette boucle peut devenir infinie. Explique pourquoi et corrige-la.

Règle d'or : ne jamais comparer des flottants avec == ou != dans une boucle.

Exercice 5 — Nombre aléatoire entre min et max

Écris une fonction random(min, max) qui retourne un nombre flottant aléatoire entre min (inclus) et max (exclus).

Formule : min + Math.random() * (max - min)

Exercice 6 — Entier aléatoire entre min et max (inclus)

Écris une fonction randomInt(min, max) où min ET max sont inclus dans les valeurs possibles.

Utilise Math.floor(Math.random() * (max - min + 1)) + min. Le +1 est crucial pour inclure max.
