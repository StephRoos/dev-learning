5.8 — WeakMap et WeakSet

WeakMap : références "faibles"

WeakMap est comme Map, mais ses clés sont uniquement des objets, et ces références sont faibles — si l'objet n'est plus référencé ailleurs, le garbage collector peut le supprimer automatiquement.

let weakMap = new WeakMap();
let obj = { name: "test" };

weakMap.set(obj, "données associées"); // seuls les objets comme clés
weakMap.get(obj);    // "données associées"
weakMap.has(obj);    // true
weakMap.delete(obj);

// Quand obj est supprimé :
obj = null;
// → l'entrée dans weakMap disparaît automatiquement (garbage collection)
// → pas de fuite mémoire !

Limitations importantes :

  • Pas de size, pas d'itération (for..of, .keys(), etc.)
  • Uniquement : get, set, has, delete

Cas d'utilisation classiques :

  • Cache lié à la durée de vie d'un objet
  • Données additionnelles sur des objets qu'on ne contrôle pas
  • Comptage de visites/états sans empêcher le garbage collection

WeakSet

Comme Set, mais uniquement des objets, références faibles. Idéal pour : "est-ce que cet objet a été traité ?"

Fondamental : utiliser WeakMap/WeakSet quand on veut associer des données à des objets dont on ne contrôle pas le cycle de vie.

Exercice 1 — Messages non lus avec WeakSet

Utilise un WeakSet pour stocker quels messages ont été lus, sans modifier les objets message.

WeakSet est parfait pour stocker des "flags" sur des objets existants sans modifier ces objets et sans risque de fuite mémoire.

Utilise un WeakMap pour stocker la date à laquelle chaque message a été lu.
