4.6 — Chaînage optionnel ?.

Concepts clés

  • Syntaxes :
    • obj?.prop — accès à une propriété
    • obj?.[key] — accès dynamique
    • obj.method?.() — appel de méthode optionnel
  • Court-circuit : si la valeur avant ?. est null ou undefined, l'expression s'arrête immédiatement et retourne undefined (sans erreur)
  • Quand l'utiliser : pour les propriétés optionnelles (données d'API, configuration partielle)
  • Ne pas abuser : n'utilise pas ?. sur des choses qui doivent toujours exister — tu masquerais des bugs
  • Nullish coalescing ?? : se combine souvent avec ?. pour fournir une valeur par défaut
// Sans ?. — plantage si intermédiaire est null
let city = user.address.city; // TypeError si address est undefined

// Avec ?. — retourne undefined silencieusement
let city = user?.address?.city;

// Les trois formes
let name = user?.name;                    // propriété
let key = "name";
let name2 = user?.[key];                  // bracket notation
let result = user?.greet?.();             // méthode optionnelle
let first = arr?.[0];                     // tableau optionnel

// Combiné avec ?? pour valeur par défaut
let city = user?.address?.city ?? "Ville inconnue";

B3 — Combiner ?? et ?.

Crée une fonction getConfig(config, path, defaultValue) qui accède à une propriété imbriquée décrite par un chemin en string (type "server.host.port") et retourne la valeur ou defaultValue si absente. 

// Ta réponse ici

// Accès à une propriété par chemin string (sans ?. natif — exercice de logique)
function getConfig(config, path, defaultValue) {
    // path est une string comme "server.host.port"
    // Divise par "." et parcours l'objet niveau par niveau
    // Retourne defaultValue si on rencontre null/undefined à un moment

    let keys = path.split(".");
    let current = config;

    for (let key of keys) {
        // Si current est null ou undefined, retourner defaultValue

        current = current[key];
    }

    return current ?? defaultValue;
}

let config = {
    server: {
        host: "localhost",
        port: 3000
    },
    database: {
        name: "mydb"
        // pas de password défini
    }
};

console.log(getConfig(config, "server.port", 8080));      // 3000
console.log(getConfig(config, "server.host", "unknown")); // "localhost"
console.log(getConfig(config, "database.password", "secret")); // "secret" (défaut)
console.log(getConfig(config, "cache.ttl", 3600));         // 3600 (défaut)

Ce pattern "getConfig avec chemin" est très courant dans les frameworks (Lodash _.get, Vue, etc.). C'est l'équivalent programmatique de ?..
