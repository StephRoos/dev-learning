4.3 — Ramasse-miettes (Garbage Collection)

Concepts clés

  • Accessibilité : un objet est conservé en mémoire s'il est accessible depuis une racine
  • Racines : variables globales, pile d'appel courante, registres CPU
  • Mark-and-sweep : le GC marque toutes les racines, puis tout ce qui est accessible depuis elles. Tout ce qui reste non-marqué est supprimé.
  • Îlots inaccessibles : des objets qui se référencent mutuellement mais ne sont plus accessibles depuis les racines → collectés
  • Tu ne contrôles pas le GC : il tourne automatiquement, pas de free() en JS
// L'objet est accessible → conservé en mémoire
let user = { name: "Alice" };

// La référence est supprimée → l'objet peut être collecté
user = null;

// Exemple d'îlot inaccessible
function createIsland() {
    let a = { name: "a" };
    let b = { name: "b" };
    a.partner = b;  // a référence b
    b.partner = a;  // b référence a
    // a et b se référencent mutuellement
    // mais personne d'externe ne les référence
    // → les deux seront collectés à la sortie de la fonction
}
createIsland();

B3 — WeakMap pour éviter les fuites

WeakMap est une Map spéciale où les clés sont des objets et les références sont "faibles" : si l'objet clé est collecté, l'entrée WeakMap disparaît automatiquement. Très utile pour les caches ! 

// WeakMap : les références vers les clés n'empêchent pas le GC
let cache = new WeakMap();

function processUser(user) {
    // Si on a déjà traité cet utilisateur, retourner le résultat en cache
    if (cache.has(user)) {
        console.log("Cache hit !");
        return cache.get(user);
    }

    // Sinon, calculer et mettre en cache
    let result = {
        displayName: user.firstName + " " + user.lastName,
        initials: user.firstName[0] + user.lastName[0]
    };

    cache.set(user, result);
    return result;
}

let user = { firstName: "Jean", lastName: "Dupont" };
console.log(processUser(user)); // calcule et cache
console.log(processUser(user)); // retourne depuis le cache

// Quand user est mis à null, l'entrée WeakMap est automatiquement supprimée
user = null;
// Pas de fuite : le GC peut collecter l'objet user et l'entrée WeakMap
console.log("WeakMap ne retient pas les objets orphelins !");

// Contrairement à Map normale :
let normalCache = new Map();
let user2 = { name: "Alice" };
normalCache.set(user2, "données");
user2 = null; // FUITE : normalCache garde encore la référence !
console.log(normalCache.size); // 1 — toujours en mémoire
