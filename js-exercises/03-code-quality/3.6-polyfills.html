3.6 — Polyfills et Transpilers

Différence fondamentale

Transpiler (ex: Babel) Polyfill (ex: core-js)
Quoi Convertit la syntaxe ES6+ en ES5 Ajoute des fonctionnalités manquantes au runtime
Exemple arrow function → function classique Array.includes pour vieux navigateurs
Quand À la compilation / build Au chargement de la page
Résoud Problèmes de syntaxe Problèmes de méthodes/API manquantes

Règle simple : transpiler ou polyfill ?

  • TRANSPILER Si tu peux réécrire la syntaxe à la main : arrow, template literals, destructuring, let/const, classes...
  • POLYFILL Si c'est une méthode ou une API qui n'existait pas : Array.includes, Object.assign, Promise, fetch...
  • LES DEUX Pour les classes ES6 : Babel transpile la syntaxe + polyfill pour les méthodes d'instance éventuelles

Exercice 3 — Écrire un polyfill pour Array.includes

Instructions

Un polyfill suit toujours ce pattern :

  1. Vérifier si la méthode existe déjà (pour ne pas écraser la version native)
  2. Si non, l'ajouter sur le prototype
// Structure d'un polyfill :
if (!Array.prototype.includes) {
    Array.prototype.includes = function(valeurCherchee) {
        // Implémentation ici
    };
}

Comportement attendu :

  • [1, 2, 3].includes(2) → true
  • [1, 2, 3].includes(5) → false
  • [1, NaN, 3].includes(NaN) → true (cas spécial : NaN !== NaN en JS !)

Piste pour NaN : utilise Number.isNaN(valeur) pour détecter NaN, car NaN === NaN est false en JavaScript.

Bonus 1 — Polyfill pour String.prototype.startsWith

'hello'.startsWith('he') → true

'hello'.startsWith('lo') → false

Note le deuxième paramètre optionnel : position (défaut : 0).
