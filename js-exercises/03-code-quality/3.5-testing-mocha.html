3.5 — Tests automatisés avec Mocha

Concepts clés

  • describe(nom, fn) : groupe des tests liés (une suite)
  • it(description, fn) : un test individuel (un cas)
  • assert.equal(actuel, attendu) : vérifie l'égalité stricte
  • assert.throws(fn) : vérifie qu'une fonction lève une erreur
  • before / after : exécutés une fois avant/après la suite
  • beforeEach / afterEach : exécutés avant/après chaque test
  • TDD : Test-Driven Development — écrire les tests AVANT le code

Structure d'un test Mocha

describe('maFonction', function() {

    it('retourne X pour l\'entrée Y', function() {
        assert.equal(maFonction(Y), X);
    });

    it('lève une erreur si paramètre invalide', function() {
        assert.throws(() => maFonction(invalide), Error);
    });

});

Règle des 3A : Arrange (préparer), Act (agir), Assert (vérifier).

Bonus 2 — TDD : tester avant de coder capitalize(str)

Méthode TDD :

  1. Les tests sont déjà écrits (regarde dans le script)
  2. Lance la page → les tests échouent (normal)
  3. Implémente capitalize(str) pour que les tests passent
  4. Relance → les tests doivent être verts

capitalize('hello') → 'Hello' (première lettre en majuscule, reste inchangé)
