3.2 — Style de codage

Concepts clés

  • Indentation : 2 ou 4 espaces (jamais de mélange tabs/espaces)
  • Accolades : style égyptien — accolade ouvrante sur la même ligne
  • Longueur de ligne : maximum 80-120 caractères par ligne
  • Nommage camelCase : calculerTotal, pas calculer_total ni CalculerTotal
  • Noms explicites : nombreDUtilisateurs plutôt que n ou num
  • Point-virgules : toujours en fin d'instruction (style courant)
  • Espaces autour des opérateurs : a + b, pas a+b
  • Une responsabilité par fonction : une fonction = une chose

Règles de style — mémo visuel

// MAUVAIS style
if(x){
let y=x*2
if(y>10){return y}else{return 0}}

// BON style
if (x) {
  let y = x * 2;
  if (y > 10) {
    return y;
  } else {
    return 0;
  }
}

Note : l'accolade ouvrante est sur la même ligne que le if — c'est le style JavaScript standard.

Instructions

function slugify(texte) {
    return texte
        .toLowerCase()
        .normalize('NFD')
        .replace(/[\u0300-\u036f]/g, '')
        .replace(/[^a-z0-9\s-]/g, '')
        .trim()
        .replace(/\s+/g, '-');
}

function paginer(items, page, parPage) {
    let debut = (page - 1) * parPage;
    let fin = debut + parPage;
    return items.slice(debut, fin);
}

function estAdulte(dateNaissance) {
    let maintenant = new Date();
    let age = maintenant.getFullYear() - dateNaissance.getFullYear();
    let moisDiff = maintenant.getMonth() - dateNaissance.getMonth();
    if (moisDiff < 0 || (moisDiff === 0 && maintenant.getDate() < dateNaissance.getDate())) {
        age--;
    }
    return age >= 18;
}
